# 科研人员认知风格量表 (RCSS)
## Researcher Cognitive Style Scale

**理论维度：** 横向整合 (Integration) vs 垂直深度 (Depth)  
**测量目标：** 评估科研工作者在认知策略上倾向于跨学科系统思维还是单一领域深耕模式  
**题项数量：** 8 题  
**计分方式：** 7 分制李克特量表 (Likert Scale)

---

## 指导语

请根据您在进行科研活动时的**真实倾向**和**日常习惯**，对以下陈述进行评分。  
请选择最符合您实际情况的数字：

**1 = 完全不同意 / 完全不符合**  
**2 = 不同意 / 不符合**  
**3 = 有点不同意 / 不太符合**  
**4 = 中立 / 不确定**  
**5 = 有点同意 / 比较符合**  
**6 = 同意 / 符合**  
**7 = 完全同意 / 完全符合**

## 计分与计算方法

### 步骤 1：计算维度总分
- **横向整合分 (I)** = A1 + A2 + A3 + A4
  - *得分范围：4–28 分*
- **垂直深度分 (D)** = B1 + B2 + B3 + B4  
  - *得分范围：4–28 分*

### 步骤 2：计算认知风格指数 (CSI)
**公式：**
```
CSI = (A1 + A2 + A3 + A4) - (B1 + B2 + B3 + B4)
```

**或等价计算：**
```
CSI = I - D
```

**结果范围：** -24 到 +24

## 理论背景与应用建议

### 理论基础
本量表基于**认知风格理论** (Cognitive Style Theory) 和**知识生产模式** (Mode 1 vs Mode 2 Knowledge Production) 构建：

- **横向整合**对应 Gibbons 等人的 "Mode 2" 知识生产（跨学科、应用导向、异质性）
- **垂直深度**对应传统的 "Mode 1" 知识生产（学科内部、基础研究、同质性）

### 应用场景

1. **职业发展规划**
   - 高分者适合向交叉学科、技术管理或创业方向转型
   - 低分者适合在技术深度要求极高的岗位（如算法工程师、核心实验科学家）长期发展

2. **团队组建参考**
   - 理想科研团队应包含不同认知风格的成员：整合型负责方向把握和对外合作，深度型负责技术攻坚和质量把控

3. **学术写作策略**
   - 整合型学者更适合发大领域综述、提出新范式、构建理论框架
   - 深度型学者更适合发表技术性极强的方法学论文或高精度实验数据

4. **个人发展预警**
   - 若长期感到"什么都懂一点但都不精"（高 I 低 D），建议进行刻意练习以建立技术护城河
   - 若长期困在细节中难以产出系统性成果（低 I 高 D），建议培养"直升机视角"能力
