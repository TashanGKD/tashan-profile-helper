# Phase 4：画像审核

## 前置检查

读取 `profiles/[姓名].md`，了解当前填写情况：
- 哪些字段已有实测数据
- 哪些字段是 AI 推断（标注「AI推断」的）
- 哪些字段尚未填写

## 展示格式

以结构化、易读的方式展示画像，按以下顺序：

### 1. 基础身份摘要（一行一项）

### 2. 能力雷达（文字版，6个维度评分）

### 3. 当前需求摘要
- 列出主要时间占用事项（1-3条）
- 列出核心难点（1-3条）
- 列出最想改变的一件事
- 若该章节为空，提示用户：「当前需求尚未填写，建议补充以获得更有针对性的发展建议。」

### 4. 认知风格结论
- 呈现 CSI 数值和类型名称
- 一句话特征描述
- 若为推断，加粗标注 **（AI推断，建议后续用量表确认）**

### 5. 动机结构摘要
- 展示 7 个维度分数（用简单条形图文字表示）
- 展示 RAI 综合指数和解读
- 若为推断，加粗标注

### 6. 人格特征摘要
- 展示 5 个维度分数和水平标签
- 若为推断，加粗标注

### 7. AI 综合解读（若尚未生成，现在生成）
- 核心驱动模式（2-3句）
- 发展风险提示（2-3条）
- 适合发展路径（1-2个方向）

---

## 审核交互流程

展示完毕后，逐维度询问用户：

```
以上是你的完整画像。请逐项告诉我：

1. 基础身份信息是否准确？
2. 能力自评是否反映了你的实际情况？
3. 当前需求描述是否符合你现在的真实状态？
4. 认知风格描述（[类型名]）符合你的感受吗？
5. 动机结构描述是否与你的实际体验一致？
6. 人格特征描述是否准确？
7. 综合解读中有没有让你觉得「不对」的地方？

你可以直接说「第3点不对，我认为...」，我会立即更新。
```

## 处理用户反馈

| 用户反馈类型 | 处理方式 |
|:---|:---|
| 「准确」/ 「基本准确」 | 在审核记录中标记该维度为「用户确认」，更新 `review_done` |
| 「不准确，修改为…」 | 读取 `update-profile/SKILL.md` 更新对应字段 |
| 「我想重新填某个量表」 | 读取对应量表 Skill，重新施测后覆盖数据 |
| 「某个维度我不确定」 | 记录为「待确认」，提示可后续补充 |

## 完成审核后操作

1. 在 `## 八、审核记录` 中写入本次审核日期、反馈摘要和处理结果
2. 将采集阶段更新为 `review_done`
3. 更新 `最后更新` 日期
4. 告知用户：

```
画像审核完成！你的科研发展画像已建立。

后续你可以随时：
- 说「补充/修改[某个字段]」更新画像内容
- 说「重新填写[量表名]」用新数据更新某个维度
- 说「查看画像」获取最新版本

画像文件保存在：profiles/[姓名].md
```
